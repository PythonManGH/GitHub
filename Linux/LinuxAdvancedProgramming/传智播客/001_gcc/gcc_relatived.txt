1. gcc工作流程
	预处理：-E
		宏替换，头文件展开，注释去掉，xxx.c -> xxx.i(这里还是可以把它理解成.c文件) 
	编译：-s (这一步最浪费时间)
		xxx.i -> xxx.s (也就是生成了汇编文件)
	汇编：-c
		xxx.s -> xxx.o (二进制文件)
	链接
		xxx.o -> xxx (可执行文件)


	** 上面这四步不是都由gcc完成的 **
		预处理：由gcc调用cpp(预处理器)完成
		编译：由gcc完成
		汇编：由gcc调用as(汇编器)完成
		链接：由gcc调用ld(链接器)完成

2. gcc常用参数
	-v/--version : 显示gcc版本
	-I : 编译的时候指定头文件的路径 可以指定相对路径
	-c : 得到.o文件
	-o : 指定生成的文件的名字
	-g : gdb调试的时候需要加上
	-D : 在编译的时候指定宏 
	-Wall : 添加警告信息
	-On : 优化代码，n是优化级别(1,2,3)
